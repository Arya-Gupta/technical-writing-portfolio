Documentation Contributions

AsyncAPI - Grammar Style Guide

This style guide defines standards for AsyncAPI documentation for grammar across topics such as abbreviations and acronyms, active voice, capitalization, spelling, and verb tense.

AsyncAPI - Fixed grammatical errors in React render engine documentation

Previously, the documentation for React render engine had some grammatical mistakes. I have contributed a fix for these.

AsyncAPI - Version update to 2.6.0

Previously, the AsyncAPI version on the Parsers page was represented incorrectly. I have updated this to the correct version.

Processing Foundation - Updated documentation for compound assignment operators to improve consistency

Previously, the documentation for compound assignment operators for Processing 4 software was inconsistent. This included operators such as addassign, divideassign, subtractassign, multiplyassign. I have worked on updating the name, syntax and description of these operators.

Processing Foundation - Rewrote the definition for lerpColor() function

In Processing 4, the lerpColor() function calculates a new color that is a blend of two other colors. The word 'increment' used in the documentation was confusing to users because it normally means 'an amount to be added'. I have rewritten the definition for this function.

Processing Foundation - Documented new menu items added on migration from Processing 3 to Processing 4

On migration from Processing 3 to Processing 4, a few of the menu items have been removed and a few have been added. I have documented these new menu items for users of the new Processing 4 software.
